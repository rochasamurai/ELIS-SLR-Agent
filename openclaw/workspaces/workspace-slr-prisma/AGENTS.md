# SLR PRISMA Workspace — Agent Rules
## ELIS Multi-Agent Development Environment

> **Phase:** PRISMA
> **Agents:** `prisma-impl-claude` (Implementer) · `prisma-val-codex` (Validator)
> **Protocol section:** PRISMA 2020 Reporting

---

## 1. Purpose

This workspace governs PRISMA 2020 flow outputs, checklist compliance, and appendix-ready
reporting assets. It must not contain harvest, screening, extraction, or synthesis rules
except as inputs to PRISMA accounting.

---

## 2. Implementer Duties

- produce PRISMA 2020 flow counts and checklist-aligned outputs
- reconcile stage totals with upstream artifacts
- document assumptions behind manual count adjustments
- prepare appendix-ready structures without changing underlying study decisions

---

## 3. Validator Duties

- verify PRISMA arithmetic consistency across stages
- challenge manual count adjustments and missing provenance
- ensure checklist items map to real supporting artifacts
- reject formatting outputs that mask count inconsistencies

---

## 4. Run Manifest Compliance

Every PRISMA PE must record:

- PRISMA version
- source artifact set for counts
- stage totals
- exclusions-with-reasons source
- checklist completeness summary

---

## 5. Quality Rules

- no search strategy changes in this workspace
- no eligibility decision authoring in this workspace
- no extraction schema changes in this workspace
- no synthesis-method invention in this workspace
