01 — Foundational Principle

AI executes. The protocol governs.

The ELIS Multi-AI Agent Server is not a chatbot or exploratory sandbox. Every agent action traces to a specific section of the ELIS 2025 Protocol. The deterministic/probabilistic authority boundary is architecturally enforced.

Core Design Contract

AI assists reasoning. Validation governs acceptance. The researcher retains final authority over all Strategic and Interpretative outputs.

1

Protocol-driven: Every agent action maps to a protocol section (§3.2–§3.8). No agent operates outside its assigned scope.

2

Continuously executing: Weekly incremental harvest keeps the Living Systematic Review current without researcher overhead.

3

Audit-complete: Every run produces a validated run_manifest.json — sufficient for PROSPERO registration and peer review.

03 — Governance & CI Enforcement

Governance is mechanised, not advisory

Ten architectural invariants are enforced through CI gates, branch protection, and PM orchestration. Three mechanisms are unique to the research pipeline: IRR enforcement, the PI Review Gate, and PRISMA arithmetic validation.

Research Pipeline Controls

✕
IRR Enforcement — Blocking Gate

Cohen's Kappa < 0.60 is a CI blocking finding. The PE cannot resubmit until bias analysis is completed, eligibility criteria are clarified, and the disagreement cohort is re-screened.

PRISMA Arithmetic — CI Gate

Σ records identified − duplicates − exclusions = studies included. Any discrepancy blocks the PRISMA PE. Zero manual reconciliation.

✓
PI Review Gate — Strategic Outputs

Synthesis and policy implication outputs are quarantined until explicit PI approval is recorded in the run manifest. Outputs cannot enter any authoritative count without approval.

10 Architecture Invariants

1

CI gate on every domain PE. run_manifest.schema.json mandatory.

2

Branch protection + PM-only assignment authority.

3

Docker secrets + truffleHog CI scan on every PR.

4

Gate 2 blocked without REVIEW_PE_<N>.md PASS.

5
No model change without PE

Model onboarding protocol §6.5. CHANGELOG entry required.

6–10
Reproducibility · Isolation · Assignment · Evidence · Drift

All run artifacts reproducible from manifest. Repo never mounted in container. Validator cannot self-start. Every PASS evidence-backed. No silent model drift.
